Using Terminal

Terminal is an app for advanced users and developers that lets you communicate with the Mac operating system using a command line interface (CLI). You enter commands and scripts (called shell scripts) to perform tasks on your Mac. [Source]

The biggest interaction the API documentation writer has with Terminal is opening it and running sets of commands. (Though, if we're being honest, that's really all any user does with Terminal.)

In-depth info

Here's a list of useful information about this topic.

Is there a shortcut way to navigate to and open a folder in Termainal (Mac only)?

On a Mac, you don't always have to run the list (`ls`) and change directory (`cd`) commands to navigate in Terminal.

Here's a shortcut navigation trick (for Macs):

  1. Open Finder.
  2. Drill down to the folder where you want to open Terminal. 
  3. Right-click that folder. 
  4. Select New Terminal at Folder. A Terminal window opens at the folder location. 
  5. Enter commands in Terminal.

What are common commands for OpenAPI documentation?

Here's a table of commonly used commands in OpenAPI documentation.

Description

Example: openapi preview-docs Supplier_Feeds_API_Documentation_Guide.yaml

For the main YAML file (required part of documentation process): Render the main YAML file throughout the documentation process to ensure changes do not break the file.

spectral lint --ruleset <file path to the folder containing spectral.yaml>/spectral.yaml <fileNameOfMainYamlFile>.yaml

Replace orange text and brackets with text, as instructed in the orange text.

Example: spectral lint --ruleset /Users/j0k05pj/github/_linting/spectral.yaml Supplier_OnRequestReports_API_Documentation_Guide.yaml

Runs a linter on the selected file to call out errors. 

Open the terminal at the folder containing the needed file (generally, the main YAML file). 

Run the command. 

This command is required in the documentation process. Address errors called out in the linting results. 

redoc-cli build -o ../docs/index.html <file name of main YAML file>.yaml

Replace orange text and brackets with text, as instructed in the orange text.

Example: redoc-cli build -o ../docs/index.html Supplier_Feeds_API_Documentation_Guide.yaml

Output the index.html file. 

This command and associated file are required in the documentation process. The internal review site if built off the index.html file (the file generated by this command).

By default, the file is placed in the /docs folder. 

redoc-cli build -o ../docs/<file name to be output>.html <file name of main YAML file>.yaml

Replace orange text and brackets with text, as instructed in the orange text.

Example: redoc-cli build -o ../docs/Supplier_Feeds_API_Documentation_Guide.html Supplier_Feeds_API_Documentation_Guide.yaml

Output an HTML file your team can share with end users. 

This command is not a required step in the documentation process. If the team asks for this HTML file, then this command outputs that file.

By default, the file is placed in the /docs folder. 

redocly bundle -o ../docs/<file name of JSON file to generate>.json -d --format json <file name of main YAML file>.yaml

Replace orange text and brackets with text, as instructed in the orange text.

Example: redocly bundle -o ../docs/Supplier_OnRequestReports_API_Documentation_Guide.json -d --format json Supplier_OnRequestReports_API_Documentation_Guide.yaml

Output the JSON file, required by Developer Portal to post to their DEV, STG, and PROD environments.

This command and associated file are required for content posted to Developer Portal in OpenAPI YAML format.

By default, the file is placed in the /docs folder. Render the file in Redocly before sending to Developer Portal. 

(For Mac users) To render the file in Redocly and then send to the Developer Portal team for posting: 

  1. In the Finder, navigate to the associated project folder. 
  2. Right click that project folder's /docs folder. 
  3. Select New Terminal at Folder. The Terminal window displays.
  4. Enter the following command in the Terminal window: openapi preview-docs <file name.extension>
    1. Replace orange text and brackets with text, as instructed in the orange text.
    2. Compare your command against this example: openapi preview-docs Supplier_Feeds_API_Documentation_Guide.json
  5. Hit Enter. 
  6. Copy the address from the Terminal window (for example, http://127.0.0.1:8080), and paste it into a browser window. 
  7. Ensure the content displays without errors.
  8. Send the file to Developer Portal by any one of the following:
    1. Email it as an attachment to the developer assigned to your project.
    2. Attach the JSON file to the Developer Portal ticket (a ticket required for all PROD pushes). 
    3. Attach the JSON file to a Slack message in #developer-portal-cms-releases, #help_developer_portal, or direct message to the developer assigned to your project.
