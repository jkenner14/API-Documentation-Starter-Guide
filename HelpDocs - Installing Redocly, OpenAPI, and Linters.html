Installing Redocly, OpenAPI, and Linters 

Redocly, OpenAPI, linters, and more work together

Redocly is the current OpenAPI rendering engine from https://redocly.com/. Developer Portal will be using this engine for their website. It must be installed properly on your workstation to run the command openapi preview-docs <fileName>.yaml. 

OpenAPI is a type of specification (https://swagger.io/specification/.) This is the important specification because it describes each tag in detail. 

YAML is a very specific specification (https://yaml.org/spec). The general YAML specification. OpenAPI is wholly a subset of the YAML specification. 

npm is a package manager for the JavaScript programming language maintained by Microsoft's npm, Inc. npm is the default package manager for the JavaScript runtime environment Node.js and is included as a recommended feature in the Node.js installer. You need it installed for Redocly and OpenAPI YAML to function. 

Node.js is a cross-platform, open-source JavaScript runtime environment that can run on Windows, Linux, Unix, macOS, and more. You need it installed for Redocly and OpenAPI YAML to function. 

Linters are tools that examine source code and call out bugs, errors, style issues, and poorly constructed segments in the programming.

In-depth info 

How do I install Redocly and OpenAPI?  

The following procedure details how to install Redocly and OpenAPI: 

  1. Test if the application is already installed. Enter the following in any command window: swagger2openapi –help. If the application is found, skip to 3. If not, proceed with the next step. 
  2. Install the application with the following. Test if the following helper applications are already installed. In a terminal window opened in any location, enter: 
    node -v 
    npm –v 
    1. If installed, the application responses with either: 
      v18.16.0; 9.5.1 
    2. If either is not installed, the application responses with either: 
      'Node' is not recognized as an internal or external command, operable program or batch file. 
      'npm' is not recognized as an internal or external command, operable program or batch file. 
    3. If at least one needs to be install: 
      1. From https://nodejs.org/en/download, download and install node-v18.16.0-x86 (or latest version). 
      2. Launch and use all defaults. 
        There may be several terminal or command windows that run. Accept any that show. 
        The install may take a moment. Ignore any errors or warnings displayed on the screen. 
        A reboot may be needed. Follow screen instructions at the command prompt. 
  3. If both are installed, consider updating NPM instead. Enter: npm install -g npm 
  4. Open a new command window and test again. Both node and npm should be valid. 
  5. After making sure NPM and node is installed, test if the following is already installed. In a terminal window opened in any location, enter: swagger2openapi –help 
    1. If you get something like this, the application needs to be installed: zsh: command not found: swagger2openapi 
    2. To install swagger2openapi, in a terminal window at any location, enter: npm install -g swagger2openapi 
    3. If the above command fails, enter: sudo npm install -g swagger2openapi 
    4. To test the installation, enter: swagger2openapi –help 
    5. If you encounter issues, skip to Installing Redocly where you will run: sudo npm i -g @redocly/cli@latest 

For the latest procedures on this topic, reference Using OpenAPI YAML.  

How do I install a linter?  

STEP ONE: Installing NPM 

You may need to install npm, a command line package manager first. 

  1. Open command window at any location. 
  2. Test if npm is already installed. Enter: npm -v 
    If a version number, such as 6.14.13, is displayed, npm is already installed. You may skip the remainder of this section. 
    If you get a message like npm is not recognized, you will need to first install NPM. 
  3. Install from https://nodejs.org/en/ 
    We recommend the Most Current Version or Recommend for Most Users version. When running the installer, always choose the defaults, usually by selecting Next. 
    After installing, the result should be like on the Mac: 
         This package has installed: 
          Node.js v15.10.0 to /usr/local/bin/node 
          npm v7.5.3 to /usr/local/bin/npm 
          Make sure that /usr/local/bin is in your $PATH. 
  4. Test the npm installation. Enter node -v, and then npm -v. Both should return a version number. 

STEP TWO: Installing Spectral Lint 

Spectral Lint application can be installed on either a Mac or Windows, using the same instructions below. If node and npm are loaded, install the application: 

  1. Open a terminal or command window from any location. 
  2. Enter: npm install -g @stoplight/spectral-cli 
  3. Or sudo npm install -g @stoplight/spectral-cli 
  4. To test the installation: spectral lint --version 
    If the application was successfully installed, a version number displays, such as 6.6.0. Otherwise, follow the screen instructions, or try to reinstall the application. 

STEP THREE: Installing the Spectral Rules File 

The rules file is a configuration file containing the rules Spectral uses for validation. It must be available for linting. 

  1. Download spectral.yaml. It is attached to this page. 
  2. The file may be located anywhere. The simplest location is the same location as the source OpenAPI YAML file. Other locations may be used. For those, see Using Files from Different Locations.  
For how to use linting, refer to the How to use a linter to check API documentation article.
For the latest procedures on this topic, reference Spectral: A Linting Application for OpenAPI files for Developers and API Documentation Writers.
